#  Приложение To Do c архитектурой VIPER

## Section_1: Какие есть требования и какие есть исходные данные. Обрисовать общий контекст ситуации.

Требуется написать на языке Swift простое приложение ToDo List с возможностью добавления, редактирования и сохранения задач для iOS в котором будут отображены навыки использования функционала языка Swift и основных фреймворков.

## Section 2: Задачи. Основная задача продемонстрировать навык написания кода с VIPER архитектурой и обязательным тестированием, потому что архитектура VIPER используется для более легкого покрытия тестами всех компонентов прилжения. Надо создать два простых экрана и context menu. Так же, при создании приложения необходимо использовать GIT для иммитации работы в команде и демонстрации навыков пошагового создания частей приложения.

## Исходные требования: 

###  1: Функционал для пользователя: 
    - Возможность отображения задач
    - Возможность добавления новой задачи
    - Возможность редактирования задачи
    - Возможность удаления задачи
    - Возможность поиска по задачам
    
###  2: Функционал приложения:
    - Использование GIT
    - Архитектура приложения: VIPER
    - Получение (загрузка по сети интернет) списка задач с использование JSON
    - Многопоточная обработка данных в приложении с использованием GCD или NSOperation
    - Сохранение данных с использование Core Data
    - Компоненты приложения должны быть покрыты Unit тестами
